Doom Virtual Health v3.0 — Minecraft 1.21.6+

v3.0 1.21.6+ 纯原版 零依赖

为 Minecraft 实体注入独立于原版血量的虚拟血量系统，自动映射到 Bossbar。支持 auto_init、高频 tick 伤害检测、自定义死亡回调、百分比血量判定、无敌控制、余数精度补偿。

架构

doom.virtual.health ← 完全独立
    │
    ▼ (依赖: scoreboard + tag)
doom.bossbar          ← 依赖 DVH，自动同步血条

快速开始

auto_init（推荐）

/summon minecraft:zombie ~ ~ ~ {data:{dvh:{max_health:"5000",health:"5000"}},NoAI:1b}
/execute as @n[tag=virtual_health_entity] run \
  function doom.bossbar:api/create {with:{name:'"Boss"',color:red,style:progress,visible:"true",targets:"@a"}}
/execute as @n[tag=virtual_health_entity] run \
  function doom.virtual.health:api/add_health {points:-500}

传统方式

/execute as @n run function doom.virtual.health:api/create {with:{max_health:2000,health:2000}}

伤害检测原理

Health = 512f（Resistance IV 保护）
↓ 受到攻击 → Health 下降
↓ Tick 检测: temp = Health × 2200，对比 baseline = 1126400
↓ delta = baseline - temp
↓ (delta × 50 + 11) / 22 → mHP
↓ mHP × damage_mult / 1000（t1+t2 防溢出）
↓ dvh.health −= mHP | 余数累加到 dvh.rem_damage
↓ Health 复位 512f
↓ dvh.health ≤ 0 → trigger_death

精度 0.001 HP（mHP），余数累计补偿截断。

damage_mult效果
1000100% 伤害（默认）
50050% 伤害
2000200% 伤害（弱点）

api/get_percentage⚡{out}导出百分比 [0,100]

scale 用法

/execute as @n[tag=vh] run \
  function doom.virtual.health:api/get_health {out:"#hp",scale:1}
  # #hp = 2000000 (mHP 原始值)
/execute as @n[tag=vh] run \
  function doom.virtual.health:api/get_health {out:"#hp",scale:1000}
  # #hp = 2000 (HP)

安装

  1. 从 Releases 下载数据包 zip
  2. 放入存档的 datapacks/ 文件夹
  3. 执行 /reload 或重进世界
  4. 聊天栏显示 [DVH] Virtual Health System Loaded. 即成功

注意事项

  • kill 参数用字节 {kill:1b}，不是布尔
  • on_death 回调中 @s[tag=virtual_health_entity] 已失效，直接用 @s
  • add_health {points:负数} 被无敌拦截；set_health 不受无敌影响
  • Bossbar api/create 重复调用 return fail
  • scale:1 = mHP，scale:1000 = HP
